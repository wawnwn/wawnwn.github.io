1 Self-supervised Learning

一些缩写是芝麻街人物名字的模型

BERT 有 340 个 million 的参数

The models become larger and larger …

  • ELMO(94M)
  • BERT(340M)
  • GPT-2(1542M)
  • Megatron(8B)
  • T5(11B)
  • Turing NLG(17B)
  • GPT-3 is 10 times larger than Turing NLG.
  • Switch Transformer(1.6T)

1.1 什么是 Self-supervised Learning

  • supervised Learning

输入是一个 x，输出是 y，如何让输出是我们想要的 y，得有我们 label 的资料。

  • self-superised Learning

自己想办法做 supervised，在没有 label 的情况下

假设我们现在只有一堆文章，没有标注，但是想办法将这些没有标注的文章分成两部分，让一部分作为模型的输入，另外一部分作为模型的标注

self-superised Learning 可以看作是一种 unsupervised learning 的方法

但是这里之所以被称为 self-superised Learning，是为了让我们指代的模型更为明确

2 BERT 简介

BERT 这个 model 是一个 transformer encoder，架构一模一样

可以做的事情是输入一排向量，输出一排向量

输入为文字为例

2.1 Masking Input

把输入文字的随机部分盖住，这个 token 的大小是自己决定的

所谓的盖住具体而言有两种做法

第一种把句子里面某一个字换成一个特殊的符号（假设 MASK）

第二种是随机把某一个字换成另外一个字

把盖住部分所对应的输出做一个 linear 的 transform，然后做 softmax，得到一个输出分布

BERT 要学的是它的输出跟盖住的部分越接近越好（minimize cross entropy）

其实就是在做一个分类的问题

2.2 Next Sentence Prediction

从资料库里拿出两个句子，在两个句子中间回家特殊的符号（SEP）代表分隔，还会在整个 sequence 的最前面加一个特殊符号（CLS）。

一整个丢到 BERT 里面

先看 CLS 对应的输出，做一个 linear 的 transform，它要做的是一个二元的分类问题，要预测的就是 yes/no，就是要预测这两个句子是不是相接的。

This approach is not helpful.

SOP: Sentence order prediction

2.3 BERT 怎么用？

可以做各式各样的任务

Downstream Tasks

其实我们需要有一些标注的资料的

BERT 在 GLUE 上的表现

2.3.1 use BERT case1

Input: sequence

output: class

比如：sentiment analysis

linear 参数是随机初始化的，BERT 的参数是 pre-train 来的（学会做填空题的 BERT）

这个任务在实际做的时候，需要有下游任务实际的标注

2.3.2 use BERT case2

Input: sequence

output: same as input（长度相同）

比如：POS tagging（词性标注）

2.3.3 use BERT case3

Input: two sequences

Output: a class

比如：Natural Language Inference（NLI）

给两个句子，一个句子是前提，一个句子是假设。机器要做的事情就是从这个前提，能不能推出这个假设，前提跟假设是否矛盾。

可以用在立场分析里面

2.3.4 use BERT case4

有限制的问答问题

答案一定出现在文章里面

输入有文章，有问题

输出两个正整数，根据两个正整数从文章里面截出来就是正确答案

唯一需要从头开始训练的东西只有两个向量 Random Initialized

这两个向量的长度跟 BERT 的输出是一模一样的

把橙色的向量跟文章对应的输出做 inner product 输出三个数值，再做 softmax 输出三个数值，得分最高的就是答案开始的位置

蓝色向量跟橙色向量做一样的事情，得到的输出就是答案结束的位置

输入的长度理论上没有，实际上是有限制的。

3 BERT 的奇闻轶事

3.1 为什么 BERT 有用呢？

输入一串文字，每个文字都有一个对应的向量（embedding），这些向量代表输入的字的意思

意思越相近的字，它们对应的向量就越接近

语言会有一字多义的问题，但是 BERT 可以考虑上下文，所以同一个字上下文不同，对应的向量也不会一样
