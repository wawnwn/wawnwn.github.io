生成式对抗网络

1 基本概念介绍

把 network 当作一个 generator 来用

特别的地方是现在 network 的输入会加上一个 random 的 variable（Z），这个 Z 是从某一个 distribution sample 出来的。

Z 每次都不一样，但是它必须有一个限制是它必须够简单（我们知道它的式子长什么样子，我们可以去从这个 distribution 去做 sample）。举例来说，这个 distribution 可以是一个 function distribution，也可以是 uniform distribution。

同样的 X 输入，随着 sample 出来的 Z 的不一样，我们输出来的值也会不一样，所以我们 output 出来的东西不再是单一一个固定的东西，而是变成了一个复杂的 distribution。

这种可以输出一个 distribution 的 network ，我们就叫它 generator。

1.1 为什么我们需要训练 generator？

例如：Video Prediction

向左转和向右转可能两种训练资料同时都存在你的训练集里面，那这样的话，机器就会认为都是对的，那它就会同时向左也向右转，那这样它就会得到一个错误的结果。

1.1.1 有什么可能性可以处理这个问题呢？

一个可能性就是让机器的输出是有机率的，让它不再是输出单一的输出，让它输出一个机率的分布。当我们给这个 network 一个 distribution 的时候，当我们给这个 network 的输入加上一个 Z 的时候，它的输出就变成了一个 distribution。它的输出就不再是固定的。我们希望训练一个 network，它可以知道说它的输出的这个分布包含了向左转的可能，也包含了向右转的可能。

1.1.2 什么时候我们会特别需要处理这种问题呢？

当我们的任务需要一点创造力的时候（我们想要找到一种 function，但是同样的输入有多种可能的输出，而这些不同的输出都是对的）

比如：

  • Drawing
  • Chatbot

generator 一个非常著名的 network 就是 generative adversarial network（GAN）

有各式各样的变形

1.2 以 Anime Face Generation 来说明

  • Unconditional generation（去掉 X）

在 GAN 里面一个特别的地方就是，除了 generator 以外，我们要多训练一个东西，这个东西叫做 discriminator。这个 discriminator 的作用是它会拿一张图片作为输入，它的输出是一个数值（Scalar）。这个 discriminator 本身也是一个 neural network，它就是一个 function。它输入一个东西就会输出一个东西。Scalar 越大就说明现在输入的这张图片越像是真实的二次元人物的图像。

1.2.1 Basic Idea of GAN

Generator 和 Discriminator

There is where the term “adversarial” comes from.

两者之间的关系：写作敌人，念作朋友

generator 和 discriminator 它们就是两个 network。network 在训练前要先初始化它的参数。假设说它们的参数都已经初始化了。接下来训练的步骤：

  1. 固定 generator，训练 discriminator。discriminator 它训练的目标是要分辨真正的二次元人物和 generator 产生出来的二次元人物它们之间的差异。
  1. 固定 discriminator，训练 generator。 跟之前训练网络的方法没有什么不同，定一个 loss，然后用 gradient descent 让 loss 越小越好。（或者说是让 discriminator 输出的值越大越好，用gradient ascent 去调 generator）
  1. 重复步骤1 和步骤2

2 理论介绍与 WGAN

2.1 训练的目标

2.1.1 怎么计算 divergence？

  • 怎么从真正的 data 里面 sample 出东西来呢？

从图库里面随机产生，随机 sample 一些图片出来，就可以得到 了

  • 怎么从 generator 里面产生一些东西出来呢？

从 normal distribution sample 出来的 vector 丢到 generator 里面，让 generator 产生一堆图片出来，那这些图片就是从 sample 出来的图片。

接下来，GAN 一整个系列的 network 就是要告诉你说，怎么在只有做 Sample 的前提之下，根本不知道 和 实际上完整的 Formulation 长什么样子，居然就估测出了 Divergence。

那这个就是要靠 Discriminator 的力量

2.1.2 Discriminator 是怎么训练出来的呢？

我们有一大堆的 Real Data，这个 Real Data 就是从 sample 出来的结果。

我们有一大堆 Generative 的 Data 就可以看作是从 sample 出来的结果。

根据 Real Data 和 Generative Data，我们可以训练一个 Discriminator。它训练的目标是看到 Real Data，就给它比较高的分数。看到 Generative Data 就给它比较低的分数，即分辨好的图跟不好的图，分辨真的图和生成的图。

这也可以写成一个式子，把它当作是一个 Optimization 的问题。

这个 Optimization 的问题是这样子的：我们要训练一个 Discriminator，这个 Discriminator 可以去 Maximize 某一个 function（Objective Function）

目标是让 越大越好，即让从 里面选出来的 y 的 越大越好，让从 里面选出来 y 的 越小越好。

早年写成这个样子是为了要把 Discriminator 跟 Binary Classification 扯上关系。事实上，这个 Objective Function 就是 Cross Entropy 乘一个负号。在训练一个 Classifier 的时候，我们就是要 minimize cross entropy。

所以我们要 maximize objective function 的时候其实又等同于 minimize cross entropy，也就等同于在训练一个 Classifier。

就是把蓝色的从 里面选出来的 真的图片作为 Class 1，把红色从 里面选出来假的图片作为 Class 2。有两个 class data，训练一个 binary 的 classifier，训练完就等同于解决了这一个 optimization 的问题。

最神奇的地方是：红色框框圈出来的东西它跟 JS Divergence 有关

实际上的推导过程可以看 https://arxiv.org/abs/1406.2661

2.2 GAN 训练的 Tips

最知名的 GAN 是 WGAN

2.2.1 JS divergence 有什么样的问题？

和 重叠的部分往往非常少。有两个理由：

  1. 来自 data 本身的特性， 和 都要产生图片，图片其实是高维空间里面的一个低维的 Manifold。二维空间中只有非常小的范围 sample 出来会是图片。 和 都是二维空间中的两条直线，除非它们刚好重合，否则它们相交的范围几乎是可以忽略的。
  2. 我们从来不知道 和 是什么样的，我们对 和 的理解来自于 sample。所以也许 和 有非常大的 overlap 的范围，但是如果我们从 和 里面 sample 出来的点不够多，也就相当于是没有重叠的。

和 几乎没有重叠这件事对 JS divergence 会造成什么问题呢？

JS divergence 的特性对于两个几乎没有重叠的特性算出来总是 。

无论实际上，这个 binary classifier 训练出来的 loss 根本没有什么意义，因为它根本没有办法看出来你的 generator 有没有越来越好。

2.2.2 Wasserstein distance

换一种衡量两个 Distribution 的相似程度的当时，换一种 divergence

假设你有两个 Distribution，一个叫 P，一个叫 Q，使用的方法就是想象在开一台推土机，P 是一堆土，Q 是要把土推过去的目的地。这个推土机把 P 这个土挪到 Q 所移动的平均距离就是 Wasserstein Distance。

把 P 重新塑造成 Q 的 plan 有很多种

不同的 moving plan 算出来的 d 就会不一样

为了让 Wasserstein distance 是一个确定值，using the “moving plan” with the smallest average distance to define the Wasserstein distance.

可以解决 JS divergence 看不出来是好是坏的问题。

2.2.3 WGAN

如果 x 是从 sample 出来的，相应的 需要大一点，如果 x 是从 sample 出来的，相应的 需要小一点。

不能是一个随便的 function，必须要是一个足够平滑的 function。
