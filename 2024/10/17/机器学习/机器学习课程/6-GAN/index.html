1 基本概念介绍

把 network 当作一个 generator 来用

特别的地方是现在 network 的输入会加上一个 random 的 variable（Z），这个 Z 是从某一个 distribution sample 出来的。

Z 每次都不一样，但是它必须有一个限制是它必须够简单（我们知道它的式子长什么样子，我们可以去从这个 distribution 去做 sample）。举例来说，这个 distribution 可以是一个 function distribution，也可以是 uniform distribution。

同样的 X 输入，随着 sample 出来的 Z 的不一样，我们输出来的值也会不一样，所以我们 output 出来的东西不再是单一一个固定的东西，而是变成了一个复杂的 distribution。

这种可以输出一个 distribution 的 network ，我们就叫它 generator。

1.1 为什么我们需要训练 generator？

例如：Video Prediction

向左转和向右转可能两种训练资料同时都存在你的训练集里面，那这样的话，机器就会认为都是对的，那它就会同时向左也向右转，那这样它就会得到一个错误的结果。

1.1.1 有什么可能性可以处理这个问题呢？

一个可能性就是让机器的输出是有机率的，让它不再是输出单一的输出，让它输出一个机率的分布。当我们给这个 network 一个 distribution 的时候，当我们给这个 network 的输入加上一个 Z 的时候，它的输出就变成了一个 distribution。它的输出就不再是固定的。我们希望训练一个 network，它可以知道说它的输出的这个分布包含了向左转的可能，也包含了向右转的可能。

1.1.2 什么时候我们会特别需要处理这种问题呢？

当我们的任务需要一点创造力的时候（我们想要找到一种 function，但是同样的输入有多种可能的输出，而这些不同的输出都是对的）

比如：

generator 一个非常著名的 network 就是 generative adversarial network（GAN）

在 GAN 里面一个特别的地方就是，除了 generator 以外，我们要多训练一个东西，这个东西叫做 discriminator。这个 discriminator 的作用是它会拿一张图片作为输入，它的输出是一个数值（Scalar）。这个 discriminator 本身也是一个 neural network，它就是一个 function。它输入一个东西就会输出一个东西。Scalar 越大就说明现在输入的这张图片越像是真实的二次元人物的图像。

2.1.2 Discriminator 是怎么训练出来的呢？

我们有一大堆的 Real Data，这个 Real Data 就是从 sample 出来的结果。

我们有一大堆 Generative 的 Data 就可以看作是从 sample 出来的结果。

根据 Real Data 和 Generative Data，我们可以训练一个 Discriminator。它训练的目标是看到 Real Data，就给它比较高的分数。看到 Generative Data 就给它比较低的分数，即分辨好的图跟不好的图，分辨真的图和生成的图。

这也可以写成一个式子，把它当作是一个 Optimization 的问题。

这个 Optimization 的问题是这样子的：我们要训练一个 Discriminator，这个 Discriminator 可以去 Maximize 某一个 function（Objective Function）

目标是让 越大越好，即让从 里面选出来的 y 的 越大越好，让从 里面选出来 y 的 越小越好。

早年写成这个样子是为了要把 Discriminator 跟 Binary Classification 扯上关系。事实上，这个 Objective Function 就是 Cross Entropy 乘一个负号。在训练一个 Classifier 的时候，我们就是要 minimize cross entropy。

所以我们要 maximize objective function 的时候其实又等同于 minimize cross entropy，也就等同于在训练一个 Classifier。

就是把蓝色的从 里面选出来的 真的图片作为 Class 1，把红色从 里面选出来假的图片作为 Class 2。有两个 class data，训练一个 binary 的 classifier，训练完就等同于解决了这一个 optimization 的问题。

3.1.1 为什么 GAN 很难被 Train 起来呢？

它有一个本质上困难的地方

Discriminator 和 Generator 各自做的事情是什么？

Discriminator 做的事情是要分辨真的图片跟产生出来的假的图片的差异

Generator 做的事情是要取产生假的图片，骗过 Discriminator

事实上，这两个 Network 是互相砥砺才能互相成长的，只要其中一者发生什么问题停止训练，另外一个人就会跟着停下训练，跟着变差。

假设你在 train Discriminator 的时候没有 train 好，那 Generator 就失去了可以进步的目标。Generator 就没有办法再进步了，Discriminator 也会跟着停下来。

3.2 GAN for Sequence Generation

train GAN 最难的是要拿 GAN 来生成文字

这个任务中，sequence to sequence 的 model 就是我们的 Generator，就是扮演过去 Transformer 里面 Decoder 的角色，负责我们要它产生的东西，比如说一段文字。

这个跟在影像上的 GAN 有什么不同呢？

可能没有太大的不同，因为接下来要去训练一个 Discriminator，这个Discriminator 要去判断说这段文字是真正的文字还是机器产生的文字，而 Decoder(Generator) 就是想办法去骗过 Discriminator，去调整 Generator 的参数，想办法让 Discriminator 觉得 Generator 产生出来的东西是真正的。

难点在于如果我们要用 gradient descent 去 train 我们的 Decoder，去让 Discriminator Output 分数越大越好是做不到的。

为什么做不到？

在计算微分的时候，所谓的 Gradient，所谓的微分，其实就是某一个参数它有变化的时候对我们的目标造成了多大的影响。假设我们改变了 Decoder 的参数，到底对 Discriminator 的输出有什么影响？

Decoder 的参数如果有一点点小小的变化，那它现在输出的这个 Distribution 也会有小小的变化，因为这个变化很小，所以它不会影响最大的 Token（我们现在在处理产生这个 Sequence 的单位，是人定的）。所以对 Discriminator 来说 Decoder 输出的东西是一样的，这样 Discriminator 输出的分数就没有改变。所以当 Decoder 的参数有一点改变的时候，Discriminator 输出是没有改变的。所以我们根本就没有办法算微分，即根本没有办法做 Gradient Descent。

遇到不能做 Gradient Descent 的问题就当作 Reinforcement Learning（RL） 的问题，硬做一下就结束了。但是这样会发生什么问题呢？

Rl 是以难 train 闻名，GAN 也是以难 train 闻名。这两个东西碰撞在一起会大炸裂。在过去一直被认为是一个很大的难题，在很长的一段时间里面，没有人可以成功地把 GAN for Sequence Generation 训练起来产生文字，通常你需要先做 Pretrain。总之，过去我们没有办法用正常的方法让 GAN 产生一段文字。

直到 Scratch GAN 的出现。

它可以直接从随机的初始化参数开始 train 它的 Generator，然后让 Generator 可以产生文字。

如何做到的？

最关键的是爆调 Hyperparameter 跟一大堆的 Tips。一开始要有一个叫做 SeqGAN-Step 的技术，没这个完全 Train 不起来。然后接下来有一个很大的 Batch Size，通常是上千。

3.4 Evaluation of Generation

怎么评估一个 GAN 产生出来的结果的好坏？

要评估一个 Generator 的好坏并不容易，最直接的方法就是人眼看产生的图片评估好坏。

有没有比较客观和自动的方法来想办法评量一个 Generator 的好坏呢？

如果针对一些特定的任务，是有办法设计一些方法的。

一个可能的做法：把 GAN 产生出来的图片丢到一个影像的分类系统里面，看它产生什么样的结果。影像分类系统输入是一张图片 y，输出是一个机率分布 。这个机率分布如果越集中就代表现在生成的图片可能越好。

但是光用这个做法是不够的，光用这个方法会被一个叫做 Mode Collapse 的问题骗过去。

3.4.1 Mode Collapse

  • 什么叫 Mode Collapse？

在 train GAN 的时候，有时候 train 着 train 着会遇到一个状况

假设蓝色的星星是真正的资料分布，红色的星星是你 Generative Model 生成的星星。你会发现 Generative Model 输出来的图片来来去去都是那几张。

  • 为什么会有 Mode Collapse 的现象发生呢？

中间的地方就是 Discriminator 的一个盲点，当 Generator 学会产生这种图片以后，它就永远都可以骗过 Discriminator。

  • 如何避免 Mode Collapse 的情况？

目前还没有一个非常好的解答。

Model 在 Generator 训练的时候，一路上都会把 train point 存下来，在 mode collapse 之前就就停下来。

但是 Mode Collapse 出现以后会很容易被看出来。

3.4.2 Mode Dropping

有一种跟 mode collapse 有点像，但是更难被侦测到的问题：Mode Dropping

产生出来的资料只有真实资料的一部分，单纯看产生出来的资料可能会觉得还不错，而且分布的多样性也足够，但其实真实资料的分布是更大的。

  • 现在要评量一下 generator 产生出来的图片多样性够不够

过去有一个做法是借助我们的 Image Classify。把 generator 产生出来的图片都丢到 Image Classify 里面，看它被判断成哪一个 Class。每张图片都会给我们一个 Distribution，把所有的 Distribution 平均起来，接下来看看平均的 distribution 长什么样子。如果平均的 distribution 非常集中，就代表现在多样性不够。

那如果另外一个 Case，不同张图片丢进去，Image Classify 产生出来的输出的分布都非常地不同，平均完以后的结果是非常平坦的。这个时候代表说也许我们的多样性是足够的。

在评估的标准上，diversity 和 quality 好像是有点互斥的。因为越集中代表 quality 越高，但是 diversity 是越平坦，分布越平均越大。但是其实 diversity 和 quality 它们评估的范围是不一样的，quality 是只看一张图片，而 diversity 看的是一堆图片。

  • Inception Score（IS）

quality 越高，diversity 越大，IS 越大

  • Fréchet Inception Distance(FID)

把产生出来的二次元人物图片，丢到 Inception Network 里面，拿进入 Softmax 之前的 Hidden Layer 的输出，把这个向量拿出来代表这张图片。

假设真实的图片和产生出来的图片它们都是 Gaussians 的 Distribution，然后去计算这两个 Gaussians Distribution 之间的 Fréchet Distance。值越小越好，距离越小，代表这两组图片越接近，即产生出来的品质越高。

3.5 Conditional Generation

目前我们的 Generator 它输入的都是一个随机的分布而已，这个不见得非常有用，我们现在想要更进一步的是我们可以操控 Generator 的输出。我们给一个 Condition x，让它根据 x 跟 z 来产生 y。

这样的 conditional generation 有什么应用呢？

比如说你可以做文字对图片的生成 Text-to-image，它其实是一个 supervised learning 的问题，你需要一些 label 的 data。在这个应用里面，我们的 x 就是一段文字。

一段文字怎么输入给 Generator 呢？

以前会用 RNN 把它读过去，然后得到一个向量，再丢到 Generator。今天也许你可以把它丢到 transformer 的 encoder 里面去，把 encoder output 这些向量统统平均起来丢到 generator 里面去，用什么方法都可以。

画出来的角色取决于你 sample 到什么样的 z。

3.5.1 Conditional GAN

我们现在的 Generator 有两个输入，一个是从 Normal Distribution sample 出来的 z，另外一个是 x，也就是一段文字，那我们的 generator 会产生一张图片 y，那我们需要一个 discriminator。如果按照我们过去所学的东西，discriminator 吃一张图片 y 当作输入，输出一个数值，这个数值代表输入的图片像真实的图片的程度。然后 discriminator 和 generator 反复训练。

但是这样的训练没办法真的解 conditional GAN 的问题，因为如果我们只有 train 这个discriminator，这个 discriminator 只会看 y 当作输入的话，generator 会产生可以骗过 discriminator 的非常清晰的图片，但是跟输入完全没有任何关系。因为对 generator 来说，它只要能产生非常清晰的图片就可以骗过 discriminator 了，它何必要去管 input 文字叙述是什么呢？discriminator 又不看文字的叙述。这显然不是我们要的。

所以对于 conditional GAN 我们要做有点不一样的设计。

对于 discriminator 不仅要吃 y 还要吃 condition x，discriminator 输出的数值不只是看 y 好不好，同时 y 跟 x（文字的叙述）必须是相配的。所以训练 discriminator 需要文字跟影像成对的资料。所以 conditional GAN 训练的资料是需要 pair 的 data 的，是需要有标注的资料。

光告诉 discriminator 好的状况是不够的，还需要加上一种不好的状况是已经产生好的图片，但是文字叙述配不上的状况。

4.2 Cycle GAN

在 Cycle GAN 里面我们会 train 两个 generator。

第一个 generator 它的工作是把 x domain 的图变成 y domain 的图。

第二个 generator 它的工作是看到一张 Y domain 的图，把它还原回 X domain的图。

经过两次转换以后，输入跟输出要越接近越好。两张图片之间的距离越近越好。

所以现在有三个 network。

cycle GAN 可以强迫第一个 genertor 的输出跟输入有一点关系，但是这个关系到底是不是我们想要的不一定，关系可能会很奇怪，无法保证输入和输出的东西看起来很像。

但其实这种情况不太会出现，这个问题不会很大。

Cycle GAN 可以是双向的

在训练的时候，可以同时做另外一个方向的训练

  • 下面三种没有什么不同，几乎是在同一时间内提出来相同的想法。
  • StarGAN

Cycle GAN 只能在两种风格间做转换，StarGAN 可以在多种风格间做转换。
